Getting started with Spec2JIRA for Confluence and Jira

Setup time: ~30 minutes. You'll need a server with an NVIDIA GPU and Docker installed.

Prerequisites

RequirementDetails
Atlassian CloudConfluence Cloud + Jira Cloud instance
ServerLinux server (Ubuntu 22.04+ recommended)
GPUNVIDIA GPU with 16GB+ VRAM (e.g., T4, L4, A10, RTX 4090)
DockerDocker CE with NVIDIA Container Toolkit
NetworkHTTPS endpoint reachable from Atlassian Cloud
Storage~30GB free disk space (model + Docker image)

1. Install the Forge App

Install Spec2JIRA from the Atlassian Marketplace. The app will appear as a panel on your Confluence pages and connect to your self-hosted backend.

2. Deploy the Backend

Pull the Docker image and start the container:

# Pull the latest image
docker pull spec2jira/backend:latest

# Create configuration
mkdir -p spec2jira && cd spec2jira
cp docker-compose.yml.example docker-compose.yml

# Generate an API key
echo "API_KEY=$(openssl rand -hex 32)" > .env

# Start the backend
docker compose up -d

3. Download the AI Model

The backend requires the Qwen 2.5 14B GGUF model (~9GB). Download it to your server:

Option A: Direct download (recommended)

# Download model to ./models/base/
./download_model.sh

Option B: Manual download (restricted networks)

On a machine with internet access, download from HuggingFace:

# On a machine with internet access
wget https://huggingface.co/Qwen/Qwen2.5-14B-Instruct-GGUF/resolve/main/qwen2.5-14b-instruct-q4_k_m.gguf

# Copy to your server
scp qwen2.5-14b-instruct-q4_k_m.gguf user@your-server:spec2jira/models/base/
Note: Some corporate networks may restrict downloads from HuggingFace. In that case, use Option B from an unrestricted machine and transfer the file via SCP or USB.

4. Configure the Connection

4.1 Set up HTTPS

Your backend must be accessible via HTTPS. We recommend nginx with Let's Encrypt:

# Install certbot and get a certificate
sudo certbot --nginx -d api.yourdomain.com

4.2 Connect Forge App to Backend

  1. Open any Confluence page
  2. Click the Spec2JIRA panel → Settings
  3. Enter your backend URL (e.g., https://api.yourdomain.com)
  4. Enter the API key from your .env file
  5. Click Test Connection to verify

5. First Run

  1. Navigate to a Confluence page with a specification document
  2. Click Generate Breakdown in the Spec2JIRA panel
  3. Wait for the AI pipeline to process (~5-10 minutes for typical documents)
  4. Review and edit the generated breakdown in the interactive editor
  5. Click Push to Jira to create the work items
First run note: The initial generation takes longer as the model loads into GPU memory. Subsequent runs are faster.

GPU Compatibility

The Docker image is built for CUDA 12.8. If your GPU requires a different CUDA architecture, rebuild with:

docker compose build --build-arg CUDA_ARCH=<your_arch>

Troubleshooting

Connection test fails

Verify your backend is accessible: curl -I https://api.yourdomain.com/health. Check that your firewall allows inbound HTTPS (port 443) and that nginx is proxying to port 8000.

Generation timeout

Large specifications (10,000+ words) may take 15-20 minutes. The Forge app will poll for completion. If it consistently times out, check GPU memory with nvidia-smi and Docker logs with docker compose logs -f.

Out of GPU memory

The 14B Q4 model requires ~8GB VRAM at rest and peaks at ~15GB during generation. Ensure no other GPU workloads are running. If using a 16GB GPU (T4), close other CUDA processes before generating.
